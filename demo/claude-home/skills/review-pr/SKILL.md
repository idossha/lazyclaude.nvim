# PR Review

When reviewing a pull request:
1. Fetch the PR diff using `gh pr diff <number>`
2. Check for code quality, security, and test coverage
3. Verify the PR description matches the changes
4. Leave constructive comments on specific lines
5. Approve, request changes, or comment with summary
