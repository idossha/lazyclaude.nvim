# Git Commit

When asked to commit:
1. Run `git status` and `git diff --staged` to understand changes
2. Draft a conventional commit message (feat/fix/refactor/docs/test)
3. Keep the subject line under 72 characters
4. Add a body if the change is non-trivial
5. Create the commit
